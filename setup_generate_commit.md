# 🤖 How To Setup AI Generated Commits (files provided in this repo)
***

   ##### 📌 <u>**Please Note:**</u> The files come preset to use Gemini. If you want to use another AI provider, please see the instructions at the end of this file.

 🔹 Alternatively, if you don't want commit messages to be auto generated by AI, simply ignore the files or delete them from your local repo.



<br>

### ⚙️ Steps To Reactivate:</h2></summary>
<hr style="border: 0.3px solid green;"></hr>
<br>

#### 1. Create a basic package.json (if your new project doesn't have one yet)
```
npm init -y
```

#### 2. Install the two required packages
```
npm install dotenv
npm install --save-dev husky
```

#### 3. Tell Git to use your copied .husky folder for its hooks
```
git config core.hooksPath .husky
```

#### 4. Restore the file's permission to execute
```
chmod +x .husky/prepare-commit-msg
```
<br>

### 🔄 How to Swap AI Providers:
<hr style="border: 0.3px solid cyan;"></hr>
<br>

Because the hook is a custom script, it is completely vendor-agnostic. You can switch models in minutes by swapping out the API Endpoint URL and the Request Payload Structure in your `generate-commit.js` file.

### Option A: Groq
Point the script to Groq's fast `llama-3.3-70b-versatile` model.

**1. Swap the environment variable at the top of your file:**
```javascript
const apiKey = process.env.GROQ_API_KEY;
```

**2. Replace the API Call (Step 6 & 7 inside `generate-commit.js`) with Groq's structure:**
```javascript
	try {
		const controller = new AbortController();
		const timeoutId = setTimeout(() => controller.abort(), 30000);

		const response = await fetch(`[https://api.groq.com/openai/v1/chat/completions](https://api.groq.com/openai/v1/chat/completions)`, {
			method: 'POST',
			headers: {
				'Content-Type': 'application/json',
				'Authorization': `Bearer ${apiKey}`
			},
			body: JSON.stringify({
				model: "llama-3.3-70b-versatile",
				messages: [{ role: "user", content: prompt }],
				temperature: 0.2
			}),
			signal: controller.signal
		});

		clearTimeout(timeoutId);
		const data = await response.json();

		if (data.error) throw new Error(data.error.message);

		let aiMessage = data.choices[0].message.content.trim();
		aiMessage = aiMessage.replace(/^```\w*\n|\n```$/g, '');

		if (commitMsgFile) {
			const currentMsg = fs.readFileSync(commitMsgFile, 'utf8');
			fs.writeFileSync(commitMsgFile, aiMessage + "\n\n" + currentMsg);
		}
		console.log(`✅ Success (via Groq)!`);

	} catch (err) {
		// Fallback logic remains exactly the same...
	}
```

### Option C: Anthropic (Claude)
Anthropic uses a different endpoint and payload structure, and requires specific version headers.

**1. Swap the environment variable at the top of your file:**
```javascript
const apiKey = process.env.ANTHROPIC_API_KEY;
```

**2. Replace the API Call (Step 6 & 7 inside `generate-commit.js`) with Anthropic's structure:**
```javascript
	try {
		const controller = new AbortController();
		const timeoutId = setTimeout(() => controller.abort(), 30000);

		const response = await fetch(`[https://api.anthropic.com/v1/messages](https://api.anthropic.com/v1/messages)`, {
			method: 'POST',
			headers: {
				'Content-Type': 'application/json',
				'x-api-key': apiKey,
				'anthropic-version': '2023-06-01'
			},
			body: JSON.stringify({
				model: "claude-3-5-sonnet-20241022",
				max_tokens: 1024,
				messages: [{ role: "user", content: prompt }]
			}),
			signal: controller.signal
		});

		clearTimeout(timeoutId);
		const data = await response.json();

		if (data.error) throw new Error(data.error.message);

		let aiMessage = data.content[0].text.trim();
		aiMessage = aiMessage.replace(/^```\w*\n|\n```$/g, '');

		if (commitMsgFile) {
			const currentMsg = fs.readFileSync(commitMsgFile, 'utf8');
			fs.writeFileSync(commitMsgFile, aiMessage + "\n\n" + currentMsg);
		}
		console.log(`✅ Success (via Anthropic)!`);

	} catch (err) {
		// Fallback logic remains exactly the same...
	}
```
